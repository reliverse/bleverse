# Security Policy

Bleverse community takes security seriously.

If you believe you've found a security vulnerability, please **do not** open a public GitHub issue.

## Reporting a Vulnerability

Please report security issues privately using **one** of the following channels:

- **GitHub Security Advisories**: open a private report in this repository's Security tab.
- **Email** (preferred for now): contact the project author at [blefnk@gmail.com](mailto:blefnk@gmail.com).

Include as much detail as possible:

- A clear description of the issue and potential impact
- Steps to reproduce (proof-of-concept if available)
- Affected components / paths
- Any relevant logs, screenshots, or environment details
- Suggested fix or mitigation (if you have one)

## Response & Disclosure

- We'll acknowledge your report as soon as possible.
- We'll investigate and take appropriate action.
- Please avoid public disclosure until a fix or mitigation is available.
- If you request credit, we'll do our best to include it in the release notes / advisory.

## Supported Versions

Security fixes are provided for the latest release / main branch.

Thank you for helping keep Bleverse safe.
